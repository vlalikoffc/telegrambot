# Введение в плагины

Плагины — это дополнительные модули для Windows‑бекенда, которые расширяют статус без доступа к Telegram API и без изменения основного кода.
Они могут:

- добавлять или заменять строки статуса;
- добавлять собственные секции;
- хранить состояние в своих файлах;
- выполнять вычисления и сетевые запросы;
- запрашивать обновление статуса через `ctx.request_update()`.

Они **не могут**:

- читать `.env` или любые секреты;
- изменять файлы внутри `windows/`;
- менять или читать приватные файлы других плагинов;
- писать в корневые папки проекта;
- использовать Telegram API напрямую.

### Философия

Плагины мощные, но контролируемые. 
Вся работа с файлами идёт **через `ctx.fs`**, а нарушения безопасности приводят к мгновенному отключению плагина.

**Пример:** попытка прочитать `.env` отключит плагин сразу.

### Только Windows

Система плагинов относится **только к Windows‑бекенду**. Linux‑версия не поддерживается и не тестировалась.

### Версионирование API

Каждый плагин обязан объявить `PLUGIN_API_VERSION`, и он должен **точно** совпадать с `CORE_PLUGIN_API_VERSION`. Иначе плагин не загрузится.
