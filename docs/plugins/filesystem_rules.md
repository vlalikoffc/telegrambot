# Правила файловой системы (ОЧЕНЬ ВАЖНО)

Файлы плагинов **строго контролируются**. Любое нарушение приводит к мгновенному отключению плагина.

## Допустимые области доступа

### 1) runtime:/ (только чтение)

Доступ к файлам Windows‑бекенда:

```
runtime:/system/
runtime:/main.py
```

**Писать сюда нельзя.**

### 2) plugins:/ (только чтение)

Можно читать файлы из корня папки `plugins/`, например:

```
plugins:/README.md
plugins:/example_plugin.py
```

**Нельзя** писать в `plugins:/` и **нельзя** заходить в чужие папки плагинов.

### 3) self:/ (полный доступ)

Единственное место, где плагин может создавать и изменять файлы:

```
self:/notes.txt
self:/data/cache.json
```

Этот путь соответствует папке `plugins/<plugin_name>/`.

## Абсолютный запрет на .env

Попытка прочитать или даже проверить существование `.env` или `.env.*`:

- немедленно вызывает исключение безопасности;
- пишет `SECURITY VIOLATION` в лог;
- отключает плагин на текущую сессию.

**Пример запрещённого кода:**

```python
ctx.fs.read_text("runtime:/.env")        # запрещено
ctx.fs.exists("plugins:/.env.production") # запрещено
```

### Важное предупреждение

Попытка обойти ограничения (например, через `..` или symlink) тоже считается нарушением.
